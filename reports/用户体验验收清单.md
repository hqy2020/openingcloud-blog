## 1. 文档说明与状态字典

### 1.1 适用范围

- 覆盖：品牌与视觉、首页七板块、分类页、文章详情页、全局组件、动画与降级、草地宠物交互、后台管理可用性、前端文案。
- 不覆盖：Obsidian 同步流程、部署/运维/备份/监控、数据库压测、CI/CD 技术流程（这些由原验收清单与运维清单负责）。
- 与原清单关系：本清单补齐 UX 维度，不替代原 API/功能验收项。

### 1.2 发布判定规则

- 阻断项：`100%` 通过才可判定 UX 维度可上线。
- 增强项：不阻断上线，但必须记录问题与优先级。

### 1.3 文档接口（固定列）

`ID | 分层 | 需求来源 | 页面/模块 | 前置条件 | 验收步骤 | 预期结果 | 证据 | 状态`

### 1.4 状态字典

- `待验`：尚未执行。
- `通过`：步骤和预期全部满足。
- `不通过`：存在影响体验的问题。
- `有风险`：当前可用但存在明显体验风险（需跟踪修复）。
- `豁免`：本轮明确不验并记录原因（仅增强项允许）。

## 4. 阻断项（前端功能，必须全部通过）

> 使用方式：逐项勾选，`- [ ]` 改为 `- [x]` 即表示通过。  
> 建议每项都附一张截图或一段录屏。

### 4.1 首页（`/`）

- [ ] 首页可以正常打开，不白屏、不报错。（UX-HOME-01）
- [ ] 首屏能看到 Hero 主标题、副标题和滚动 Slogan。（UX-HOME-02）
- [ ] 首页 7 个板块顺序正确：Hero → 人生足迹 → 高光时刻 → 旅行足迹 → 社交图谱 → 数据面板 → 联系方式。（UX-HOME-01）
- [ ] Hero 在降级场景也可用（静态图或视频可显示，不黑屏）。（UX-HOME-03）
- [ ] 人生足迹有节点信息，类型和重点节点能看出区别。（UX-HOME-05）
- [ ] 高光时刻卡片能正常展示，信息不重叠。（UX-HOME-07）
- [ ] 旅行足迹能展示地图；若地图不可用，有可读的降级卡片。（UX-HOME-08）
- [ ] 社交图谱在访客态只显示匿名信息，不出现真实姓名/链接/头像。（UX-HOME-09）
- [ ] 数据面板数字可读，桌面和手机都不拥挤。（UX-HOME-10）
- [ ] 联系方式可用：邮箱操作有反馈，GitHub 能打开新窗口。（UX-HOME-11）
- [ ] 首页在加载中/接口失败时有清晰提示，页面不会中断。（UX-HOME-12）

### 4.2 分类页（`/tech`、`/learning`、`/life`）

- [ ] 三个分类页都能打开，且风格区分明显。（UX-CATEGORY-01）
- [ ] 分类头部信息完整：标题、副标题、统计信息可读。（UX-CATEGORY-02）
- [ ] 标签筛选可点击，默认“全部”生效，手机端可横向滑动标签。（UX-CATEGORY-03）
- [ ] 排序切换（最新/阅读最多）能改变列表顺序。（UX-CATEGORY-04）
- [ ] 瀑布流为双列布局，桌面和手机都不重叠不乱序。（UX-CATEGORY-05）
- [ ] 文章卡片关键信息完整：封面、标题、日期、阅读数、阅读时长。（UX-CATEGORY-06）
- [ ] 点击文章卡片可以进入正确详情页。（UX-CATEGORY-07）
- [ ] 无匹配内容时有空状态文案，加载状态可感知。（UX-CATEGORY-08）

### 4.3 文章详情页（`/posts/:slug`）

- [ ] 文章详情页可打开，内容可读，页面不抖动。（UX-POST-01）
- [ ] 阅读进度条随滚动变化，进度连续。（UX-POST-01）
- [ ] 封面图显示正常，桌面/手机都不拉伸。（UX-POST-02）
- [ ] 文章头部信息完整：标题、日期、阅读时长/阅读量。（UX-POST-03）
- [ ] Markdown 正文样式正常：段落、代码块、链接、图片都可读。（UX-POST-04）
- [ ] 桌面端目录可用：能跳转、定位准确。（UX-POST-05）
- [ ] 文末标签和“返回分类”可用。（UX-POST-06）
- [ ] 错误 slug 或接口失败时有清晰提示，不出现空白页。（UX-POST-07）

### 4.4 全局交互与响应式

- [ ] 顶部导航固定可见，4 个菜单项高亮正确。（UX-GLOBAL-01）
- [ ] 手机端导航可展开/收起，点菜单能正确跳转。（UX-GLOBAL-03）
- [ ] 404 页面完整（文案+按钮），可一键回到主路径。（UX-GLOBAL-04）
- [ ] 全站关键图片交互可用：桌面 hover 揭示、移动端滚入揭示。（UX-GLOBAL-06）
- [ ] 响应式正常：页面无横向溢出，文字和按钮不遮挡。（UX-GLOBAL-07）
- [ ] 开启 `prefers-reduced-motion` 后，动效会降级且页面仍可用。（UX-GLOBAL-08）

### 4.5 SEO 可感知检查（前端侧）

- [ ] 首页/分类页/详情页都包含基础 Meta：title、description、og、canonical。（UX-SEO-01）
- [ ] `robots.txt` 可访问。（UX-SEO-02）
- [ ] `sitemap.xml` 可访问且格式正常。（UX-SEO-03）
- [ ] 预渲染页面源码能看到核心内容，不是纯空壳。（UX-SEO-05）
- [ ] 公开社交图谱接口和页面都不泄露敏感字段。（UX-SEO-04）

### 4.6 宠物交互（草地系统）

- [ ] 点击页面空白处会种草并有生长反馈。（UX-PET-01）
- [ ] 点击链接/按钮等可交互元素时不会误触发种草。（UX-PET-02）
- [ ] 宠物状态流转完整：`idle -> walking_to_grass -> eating -> idle`。（UX-PET-03）
- [ ] 多次点击时符合限制：最多 10 棵草，超出按先种先淘汰，点击有节流。（UX-PET-04）
- [ ] 吃草动画结束后，草会消失，宠物恢复空闲状态。（UX-PET-05）
- [ ] 低内存或减少动态场景会自动禁用宠物交互。（UX-PET-06）

### 4.7 文案与反馈（通俗可读）

- [ ] 首页主文案和 CTA 与文案库“推荐主文案”一致。（UX-COPY-04）
- [ ] 分类页标题、副标题、筛选和空状态文案语义清晰。（UX-COPY-05）
- [ ] 文章页目录/进度/返回文案可理解。（UX-COPY-06）
- [ ] 404 文案有安抚感，并给出明确回流按钮。（UX-COPY-07）
- [ ] 加载、成功、失败、无网等状态有对应提示。（UX-COPY-09）
- [ ] 页面文案不出现夸张营销词或禁用词。（UX-COPY-03）
- [ ] 手机端标题和按钮不截断、不挤压。（UX-COPY-02）

### 4.8 后台体验（如果本轮要一并验）

- [ ] 管理员登录/退出流程清晰，失败有提示。（UX-ADMIN-01）

## 6. 缺陷记录模板与反馈格式

### 6.1 缺陷记录模板（复制即可）

- 验收项：`UX-HOME-08`
- 勾选结果：`- [x]` / `- [ ]`
- 实际结论：通过 / 不通过 / 有风险 / 豁免
- 设备浏览器：例如 `iPhone 15 + Safari`
- 现象：一句话描述问题
- 期望：一句话描述预期
- 证据：截图/录屏文件名或链接
- 优先级：P0 / P1 / P2 / P3

### 6.2 回传格式（沿用编号）

- 已通过：`UX-HOME-01, UX-HOME-02, UX-CATEGORY-01`
- 有问题：`UX-POST-05`（设备 + 现象 + 证据）
- 有风险：`UX-GLOBAL-06`（原因）
- 暂不验：`UX-PET-54`（原因）

### 6.3 本轮默认

- 阻断项必须全部勾成 `- [x]` 才算 UX 可上线。
- 增强项可分批完成，但要记录未通过项和优先级。
