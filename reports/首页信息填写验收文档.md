# 首页信息填写验收文档

> 依据文档：`/Users/openingcloud/openingcloud-blog/assets/前端文字素材文档.md`、`/Users/openingcloud/openingcloud-blog/assets/博客需求文档.md`  
> 字段对齐：`/Users/openingcloud/openingcloud-blog/frontend/src/api/home.ts`

## 使用说明

- [ ] 本文档只验收“首页需要填写的信息是否齐全”，不覆盖动画和性能专项。
- [ ] 每一项勾选标准：字段已填写，且页面或接口可读到对应数据。

## 1. Hero（`hero`）

- [ ] `hero.title` 已填写（如需前端动态显示标题，需确认组件是否读取该字段）。
- [ ] `hero.subtitle` 已填写为首页副标题文案。
- [ ] `hero.slogans` 已填写 6-8 条轮播短句。
- [ ] `hero.fallback_image` 已配置为可访问图片地址。
- [ ] `hero.fallback_video` 已配置为可访问视频地址（可选）。
- [ ] Hero 对应 OG 分享图素材已准备。

## 2. 人生足迹（`timeline[]`）

- [ ] 时间线节点数量已满足目标（建议不少于 13 个）。
- [ ] 每个节点必填字段完整：`title`、`start_date`、`type`、`impact`、`sort_order`。
- [ ] 每个节点补充字段完整：`description`、`end_date`、`phase`、`tags`、`cover`、`links`。
- [ ] 至少包含 1 个 `type=reflection` 的沉淀节点。
- [ ] 关键节点已标记为 `impact=high`，普通节点已区分 `medium/low`。

## 3. 高光时刻（`highlights[]`）

- [ ] 高光阶段已录入（建议覆盖小学/中学/本科/硕士/关键事件等阶段）。
- [ ] 每个阶段必填字段完整：`title`、`sort_order`。
- [ ] 每个阶段补充字段完整：`description`、`start_date`、`end_date`。
- [ ] 每个阶段至少有 1 条 `items[]` 成就记录。
- [ ] 每条成就字段完整：`title`、`description`、`achieved_at`、`sort_order`。

## 4. 旅行足迹（`travel[]`）

- [ ] 已录入去过的省份与城市数据（至少 1 省 1 城）。
- [ ] 城市必填字段完整：`province`、`city`、`sort_order`。
- [ ] 地图定位字段已填写：`latitude`、`longitude`（用于地图散点）。
- [ ] 城市补充信息已填写：`notes`、`visited_at`、`cover`（可选但建议）。
- [ ] 每个省份 `count` 与其 `cities` 数量一致（接口聚合结果正确）。

## 5. 社交图谱（`social_graph`）

- [ ] 公开好友节点已配置：`public_label`、`stage_key`、`sort_order`。
- [ ] 各关键阶段均有可关联好友（小学/初中/高中/同济/浙大/工作/家庭）。
- [ ] 隐私字段按需维护：`name`、`avatar`、`profile_url`（仅管理员态显示）。
- [ ] 不公开好友已设置 `is_public=false`，访客态不暴露真实信息。

## 6. 数据面板（`stats`）

- [ ] 已发布文章数据可支撑首页统计（`published_posts_total` 非长期为 0）。
- [ ] 标签数据已完整维护（`tags_total` 可正确统计）。
- [ ] 阅读统计已累计（`views_total` 可用）。
- [ ] 时间线/旅行/社交/高光统计与后台数据一致。

## 7. 联系方式（`contact`）

- [ ] `contact.email` 已配置为有效邮箱地址。
- [ ] `contact.github` 已配置为可访问链接。
- [ ] 相关环境变量已确认：`PUBLIC_CONTACT_EMAIL`、`PUBLIC_GITHUB_URL`。

## 8. 首页素材准备

- [ ] 品牌 Logo 图片已准备并可用于导航/品牌区。
- [ ] Hero fallback 背景图已准备（建议 1920x1080）。
- [ ] Hero fallback 视频已准备（可选）。
- [ ] OG 分享图已准备并可用于社媒预览。
- [ ] 中国地图素材已准备（SVG/GeoJSON/ECharts 地图数据）。
- [ ] 好友头像素材已准备（管理员视角使用，可选）。
- [ ] 数据面板与联系方式图标素材已准备（可选）。
- [ ] 高光封面图已准备（3-5 张，可选）。

## 9. 最终验收

- [ ] 接口 `GET /api/home/` 返回完整结构：`hero/timeline/highlights/travel/social_graph/stats/contact`。
- [ ] 首页 7 个板块均有内容展示，无“暂无数据”占位提示。
- [ ] 移动端首页无明显文字截断、空白块或关键信息缺失。
