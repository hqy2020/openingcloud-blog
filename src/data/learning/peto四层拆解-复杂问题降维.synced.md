# PETO四层拆解 - 复杂问题降维

## 定义
> **解决不了的问题，降一个维度就能解决。** 将复杂项目按 Project → Task → Event → Operation 四层拆解，找到可立即执行的最小步骤。

## 核心洞见

> **解决不了的问题，降一个维度就能解决。**
>
> 卡在哪一层，就往下拆一层。

---

## 四层模型

```
P（Project）项目层 ← 最复杂，战略级
    ↓ 拆解
T（Task）任务层 ← 明确的子任务
    ↓ 拆解
E（Event）事件层 ← 具体的执行事件
    ↓ 拆解
O（Operation）操作层 ← 最简单，立刻能做
```

---

## 使用方法

**当你感到「这事太难了」时**：
1. 判断当前卡在哪一层
2. 往下拆一层，找到可执行的下一步
3. 完成最小操作，获得正反馈
4. 逐步向上累积

---

## 实战案例

**春招拿Offer**：
```
P（Project）：春招拿到 Java 后端 Offer
    │
    ├── T（Task）：准备简历
    │       ├── E（Event）：写阿里实习经历
    │       │       └── O（Operation）：打开简历模板，写第一句话
    │       └── E（Event）：准备 AI 项目介绍
    │
    ├── T（Task）：准备面试
    │       ├── E（Event）：刷八股文
    │       └── E（Event）：准备项目讲解
    │
    └── T（Task）：投递跟进
```

**「我不知道简历怎么写」→ 降维**：
- Task: 写阿里实习
- Event: 打开模板
- **Operation: 先写一句「负责 XX 系统的 YY 功能开发」**

---

## 关键心法

- **卡住 = 维度太高**
- **行动 = 降到能做的最小步骤**
- **完成最小步骤 = 获得动力继续**
